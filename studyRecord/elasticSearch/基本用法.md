## RESTFul API

ES 遵循RestFul API  
API基本格式: http://ip:port/<索引>/<类型>/<文档id>  
常用HTTP动词: GET/PUT/HEAD/DELETE

## 基本用法

### [索引创建](https://www.elastic.co/guide/en/elasticsearch/reference/current/indices-create-index.html)

**语法**
```
	PUT index  //索引相当于关系型数据库中的database
	{
		"settings" : {
			"number_of_shards" : 1,//分片数
			"number_of_replicas":1 //副本数
		},
		"mappings" : {
			"type1" : { //类型,相当于关系型数据库中的table
				"properties" : {
					"field1" : { //属性 相当于columns
						"type" : "text" //属性类型 
						}
				}
			}
		}
	}
```
**举例**:
```
PUT /people
{
	"settings":{
		"number_of_shards":3,
		"number_of_replicas":1
	},
	"mappings":{
		"man":{
			"properties":{
				"name":{
					"type":"text"
				},
				"age":{
					"type":"integer"
				},
				"country":{
					"type":"keyword"
				},
				"date":{
					"type":"date",
					"format":"yyyy-MM-dd HH:mm:ss||yyyy-MM-dd||epoch_millis"
				},
				"description":{
                       "type": "text"
                }
			}
		}
	}
}
```

### 修改索引(当新增一个字段后,该字段不允许进行修改)

```//添加字段
   PUT /my_index/_mapping/my_type
   {
       "my_type": {
           "properties": {
               "english_title": {
                   "type":     "string",
                   "analyzer": "english"
               }
           }
       }
   }
```

### [更新/修改数据](https://www.elastic.co/guide/en/elasticsearch/reference/current/docs-update.html)  
> 使用**_update**关键字
1. 通过部分文档更新 
    **语法**:  
    ```
        POST /<index>/<type>/<id>/_update
        {
            "doc":{
                "field":"value"
            }
        }
    ```
    
    **实例**:
    ```
        POST people/man/1/_update 
        {
          "doc": {
            "age":88,
            "name":"隔壁laowang"
          }
        }
    ```
    > 如果在请求发送前,需要被修改的值与请求提交的值相同,则忽略该请求,并返回result:noop
    ```
        {
          "_index": "people",
          "_type": "man",
          "_id": "1",
          "_version": 4,
          "result": "noop",
          "_shards": {
            "total": 0,
            "successful": 0,
            "failed": 0
          }
        }
    ```
    
2. 通过脚本更新
    
    语法:
    ```
       POST <index>/<type>/<id>/_update
       {
           "script" : {
               "source": "ctx._source.counter += params.count",
               "lang": "painless",
               "params" : {
                   "count" : 4
               }
           }
       }
    ```
    
    **举例**:
    ```
        POST people/man/3/_update
               {
                   "script" : {
                       "source": "ctx._source.age += params.age",
                       "lang": "painless",
                       "params" : {
                           "age" : 400
                       }
                   }
               }
               //result:updated
    ```
    
### 删除 (索引/文档)

使用**DELETE** 动作

```
//删除索引
DELETE /twitter
//根据id 删除document
DELETE /twitter/_doc/1
//如果使用routing来控制文档,必须提供正确的routing参数
DELETE /twitter/_doc/1?routing=kimchy
//_delete_by_query关键字
POST twitter/_delete_by_query
{
  "query": { 
    "match": {
      "message": "some message"
    }
  }
}
```

### 查询

1. 准备工作
    1. 创建索引
        ```
           PUT book
           {
             "settings": {
               "number_of_shards": 5,
               "number_of_replicas": 1
             },
             "mappings": {
               "novel": {
                 "properties": {
                   "count": {
                     "type": "integer"
                   },
                   "name": {
                     "type": "text"
                   },
                   "author": {
                     "type": "text"
                   },
                   "title": {
                     "type": "text"
                   },
                   "updateDate": {
                     "type": "date",
                     "format": "yyyy-MM-dd||yyyy-MM-dd HH:mm:ss||epoch_millis"
                   },
                   "description": {
                     "type": "text"
                   },
                   "price": {
                     "type": "integer"
                   }
                 }
               }
             }
           }
           ```
    2. 添加文档
    ```
    POST book/novel/1
    {
      "author":"jack",
      "title":"java",
      "count":10000,
      "updateDate":"2017-11-02 00:00:00",
      "description":"这本书是java; this is a java book",
      "price":188
    }
    
    POST book/novel/2
    {
      "author":"jackson",
      "title":"python",
      "count":10086,
      "updateDate":"2017-11-22 00:00:00",
      "description":"这本书是python; this is a python book",
      "price":288
    }
        
    POST book/novel/3
    {
      "author":"sara",
      "title":"SQL",
      "count":12306,
      "updateDate":"2018-01-22 00:00:00",
      "description":"这本书是SQL; this is a SQL book",
      "price":388
    }
    ```
2. 开始查询
    1. id 查询
        > GET book/novel/1
    2.  条件查询  
           ```
            GET  book/_search
                 {
                   "query":{
                     "match": {
                         "title":"java"
                     }
                   },
                   "sort":[
                   {
                      "updateDate":{"order":"desc"}
                   }],
                   "from":0,
                   "size":5
                 }
           ```
           
       3. 聚合查询,可以嵌套,求出该桶中的一些统计值,比如根据时间分组,统计每个分组内的书的平均价格
       ```
           GET book/_search
           {
             "size": 0,
             "aggs": {
               "group_by_updateDate": {
                 "terms": {
                   "field": "updateDate",
                   "size": 10
                 },
                 "aggs": {
                   "avrange_price": {
                     "avg": {
                       "field": "price"
                     }
                   }
                 }
               },
               "grada_by_count": {
                 "stats": {
                   "field": "count"
                 }
               }
             }
           }
       ```
       
       4. 高级查询
       
           分类:
           - 子条件查询:  
            - query Context:
                ```
                    
                ```
            - filter Context:只判断文档是否满足条件,返回yes|no
                ```
                    结合bool 关键字,可以对多个条件进行查询,返回bool值
                    {
                       "bool" : {
                          "must" :     [],

                          "must_not" : [],
                       }
                    }
                ```

           - 复合条件查询
